linkMerging Nodes and Relationships

Merging nodes and relationships happens with the same functions as creating nodes and relationships, by passing the appropriate params. Therefore, it is recommended that Creating Nodes and Relationships is known well.

linkMerging a single or many nodes of a Model

For merging a single node, createOne can be used with the merge param set to true.

1/* --> merge a User node and get the Instance */

2const user = await Users.createOne(

3    /* --> the properties of the User node to be merged */

4    {

5        id: '1',

6        name: 'John',

7        age: 38,

8    },

9    {

10        /* --> by setting this to true, a MERGE query instead of a CREATE one will run */

11        merge: true,

12        /* --> (optional, default true) validates the properties of the node */

13        validate: true,

14        /* --> (optional) */

15        session: null,

16    }

17);

18

19/* --> we can use the Instance as usual */

20console.log(user.name); // 'John'

For merging many nodes, createMany can be used with the merge param set to true.

1const users = await Users.createMany(

2    [

3        {

4            id: '1',

5            name: 'John',

6        },

7        {

8            id: '2',

9            name: 'Alex',

10        }

11    ],

12    {

13        /* --> by setting this to true, a MERGE query instead of a CREATE one will run */

14        merge: true,

15        /* --> (optional, default true) validates all nodes */

16        validate: true,

17        /* --> (optional) */

18        session: null,

19    }

20);

21

22console.log(usersWithOrders[0].id); // '1'

23console.log(usersWithOrders[1].bar()); // 'The name of this user is: Alex'

linkCreating/Merging nodes and merging relationships with other nodes

When automatically associating with other nodes (either by creating them or by matching them), a MERGE instead of a CREATE can be used.

The following example uses createMany, but the same interface applies on createOne and instance save (when it doesn't exist in the database).

1const usersWithOrders = await Users.createMany(

2    [

3        {

4            id: '1',

5            name: 'John',

6        },

7        {

8            id: '2',

9            name: 'Alex',

10            /* --> associate with other nodes. The appropriate key (i.e. 'RelatedNodesToAssociate') can be taken like this */

11            [Users.getRelationshipCreationKeys().RelatedNodesToAssociate]: {

12                /* --> the Orders alias will be used, as defined in the Users model */

13                Orders: {

14                    /* --> (optional) configuration regarding what aspects of the 'attributes' property to merge instead of create */

15                    attributesMergeConfig: {

16                        /* --> (optional) the created (Order) nodes will be merged, instead of created */

17                        nodes: true,

18                        /* --> (optional) the relationship between User and Order nodes will be merged, instead of created */

19                        relationship: true,

20                    },

21                    /* --> (optional) merge new nodes (as attributesMergeConfig.nodes is true) and associate with them */

22                    attributes: [

23                        /* --> creates the following 2 Order nodes, and creates a relationship with each one of them using the configuration of the Orders alias  */

24                        {

25                            id: '2'

26                        },

27                        {

28                            id: '3',

29                            items: 5,

30                            /* --> the relationship is merged (as attributesMergeConfig.relationship is true) with the following properties. The appropriate key (i.e. 'RelationshipValuesToCreate') can be taken like this */

31                            [Users.getRelationshipCreationKeys().RelationshipValuesToCreate]: {

32                                createdAt: '2020-02-02'

33                            },

34                            /* --> can create (or merge) nodes and associate them with this Order node. The alias and configuration is that of the Orders model */

35                            [Orders.getRelationshipCreationKeys().RelatedNodesToAssociate]: {

36                                /* --> the 'Critics' alias will be used, as defined in the 'Orders' model */

37                                Critics: {

38                                    attributesMergeConfig: {

39                                        /* --> by setting this to false (or omitting it in the first place), the Critics nodes will be created, not merged */

40                                        nodes: false,

41                                        /* --> by setting this to false (or omitting it in the first place), the relationship between the Orders and the Critics nodes will be created, not merged */

42                                        relationship: false,

43                                    },

44                                    attributes: [{ id: '10' }]

45                                }

46                            }

47                        }

48                    ],

49                    /* --> (optional) also associates the User node with existing Order nodes */

50                    where: [

51                        {

52                            /* --> (optional) the relationship between the created User nodes and the matched Order nodes will be merged, instead of being created */

53                            merge: true,

54                            /* --> the Where clause find matching the existing Nodes */

55                            params: {

56                                id: '3'

57                            },

58                            /* --> (optional) properties can be added to the relationship merged by matching the User node with the existing Order nodes */

59                            [Users.getRelationshipCreationKeys().RelationshipValuesToCreate]: {

60                                createdAt: '2020-01-01',

61                            }

62                        },

63                        {

64                            /* --> another object can be used for matching the User node with the Order nodes of this where independently */

65                            params: {

66                                items: 3,

67                            },

68                            /* --> the realtionship of this match can be created, not merged */

69                            merge: false,

70                        }

71                    ]

72                },

73                /* --> other aliases can be used here, to associate the User node with those of other Models */

74            }

75        }

76    ],

77    {

78        /* --> merges the root-level nodes (the Users - 'John', 'Alex') */

79        merge: true,

80        /* --> (optional, default true) validates all nodes */

81        validate: true,

82        /* --> (optional) */

83        session: null,

84    }

85);
