linkSessions

All Model and QueryRunner queries use a session in order to run. A session can be created by using the driver, or by using the getSession helper to help with running queries in the same session.

All Model queries can accept an existing session as a parameter. If it's given, every query will run in this session. If not, a new session will be created for the queries to run.

All QueryRunner queries require an existing session.

linkGetting a session from the driver

1/* --> the driver can be obtained by the Neogma instance */

2const driver = neogma.getDriver();

3/* --> use the driver to create a session */

4const session = driver.session();

5/* --> this session can be used in more than one Neogma operations */

6await Users.createOne(

7    {

8        id: '1',

9        name: 'John'

10    },

11    {

12        session

13    }

14);

15await Users.findAll({

16    session

17});

18/* --> the session can also be used in the QueryRunner. Let 'queryRunner' be a QueryRunner instance */

19await queryRunner.run('RETURN 1 = 1', {}, session); // @see [QueryRunner](./QueryRunner/Overview)

20/* --> closing the session */

21await session.close();

linkGetting a session with the getSession helper

A session can also be obtained with the getSession helper. Its first parameter is an existing session: if it's set, it will be used as-is in the callback. Else, a new one will be created. Ιts second parameter is a callback with the session as the first parameter.

After the callback is done, the session automatically closes.

If getSession calls are nested, the session will close after all of them are done.

1await getSession(

2    /* --> no session is passed, so a new one will be created */

3    null, 

4    async (session) => {

5        /* --> this session can be used in more than one Neogma operations */

6        await Users.createOne(

7            {

8                id: '1',

9                name: 'John'

10            },

11            {

12                session

13            }

14        );

15

16        /* --> let's create a function that takes a session param */

17        const myFindUser = (sessionParam) => {

18            /* --> this param is used for a getSession. So, if it already exists, it will be used. Else, a new one will be created */

19            await getSession(sessionParam, async (sessionToUse) => {

20                /* --> we can perform multiple operations with this session */

21                await Users.findAll({

22                    session: sessionToUse

23                });

24                await Users.findOne({

25                    session: sessionToUse

26                });

27                await queryRunner.run('RETURN 1 = 1', {}, sessionToUse);

28            }, driver);

29        };

30

31        /* --> no session is given, so the operations in 'myFindUser' will run in their own session */

32        await myFindUser(null);

33        /* --> the existing session is given, so the operations in 'myFindUser' will run using this existing session */

34        await myFindUser(session);

35    },

36    /* --> a neo4j driver is needed */

37    driver

38);

39/* --> at this stage, the session will close */

As we can see, this helper is useful when we have to run multiple operations in a single session, but we want to easily allow an existing session to be used.

linkGetting a transaction from a session

An existing session can be used to create a transaction.

1/* --> let 'session' be a session object, acquired by one of the above methods */

2const transaction = session.beginTransaction();

3/* --> this transaction can be used in more than one Neogma operations */

4await Users.createOne(

5    {

6        id: '1',

7        name: 'John'

8    },

9    {

10        session: transaction

11    }

12);

13await Users.findOne({

14    session: transaction

15});

16/* --> commit the transaction */

17await transaction.commit(); // or .rollback() to roll it back

linkGetting a transaction with the getTransaction helper

A transaction can also be obtained with the getTransaction helper. Its first parameter is an existing transaction or a session.

  • If it's a transaction, it will be used as-is in the callback.
  • Else, a new one will be created. If the first parameter is a session, it will be used to create the transaction. Ιts second parameter is a callback with the transaction as the first parameter.

After the callback is done, the transaction automatically gets commited and the session closes. In case an error is thrown, the transaction automatically gets rolled back.

If getTransaction calls are nested, the transaction will commit after all of them are done.

1await getTransaction(

2    /* --> no transaction is passed, so a new one will be created */

3    null,

4    async (transaction) => {

5        /* --> this transaction can be used in more than one Neogma operations */

6        await Users.createOne(

7            {

8                id: '1',

9                name: 'John'

10            },

11            {

12                session: transaction

13            }

14        );

15

16        /* --> getTransaction calls can be nested */

17        await getTransaction(

18            /* --> the existing transaction is used */

19            transaction, 

20            async (transactionToUse) => {

21                /* --> we can perform multiple operations with this transaction */

22                await Users.findAll({

23                    session: transactionToUse

24                });

25                await Users.findOne({

26                    session: transactionToUse

27                });

28                await queryRunner.run('RETURN 1 = 1', {}, transactionToUse);

29            }

30        );

31    },

32    /* --> a neo4j driver is needed */

33    driver

34);

35/* --> at this stage, the transaction will be committed */

As we can see, this helper is useful when we have to run multiple operations in a single transaction, but we want to easily allow an existing transaction to be used.

linkGetting a session or transaction with the getRunnable helper

The getRunnable helper can be used to acquire an existing transaction or session. If none is given, a new session will be created.

In case nothing is given:

1await getRunnable(

2    /* --> no runnable is passed, so a new session will be created */

3    null,

4    async (session) => {

5        await queryRunner.run('RETURN 1 = 1', {}, session);

6    },

7    /* --> a neo4j driver is needed */

8    driver

9);

In case a session is given:

1await getRunnable(

2    /* --> no runnable is passed, so a new session will be created */

3    null,

4    async (session) => {

5        await queryRunner.run('RETURN 1 = 1', {}, session);

6

7        await getRunnable(

8            /* --> a session is passed, so it will be used */

9            session,

10            async (sessionToUse) => {

11                /* --> this runs in the same session as the other query */

12                await queryRunner.run('RETURN 1 = 1', {}, sessionToUse);

13            },

14            driver

15        );

16    },

17    driver

18);

In case a transaction is given:

1/* --> let's create a transaction */

2await getTransaction(

3    null,

4    async (transaction) => {

5        await queryRunner.run('RETURN 1 = 1', {}, transaction);

6

7        await getRunnable(

8            /* --> a transaction is passed, so it will be used */

9            transaction,

10            async (transactionToUse) => {

11                /* --> this runs in the same transaction as the other query */

12                await queryRunner.run('RETURN 1 = 1', {}, transactionToUse);

13            },

14            driver

15        );

16    },

17    driver

18);

As we can see, this helper gives us flexibility when we want to run queries no matter in a session or transaction, depending on what was given in parameters.

linkUsing neogma instance wrappers for these functions

A neogma instance also provides a wrapper of those functions, with the last parameter (the driver) being omitted.

1await neogma.getSession(null, async (session) => {

2    await Users.findOne({

3        session

4    });

5});

6

7await neogma.getTransaction(null, async (transaction) => {

8    await Users.findOne({

9        session: transaction

10    });

11});

12

13await neogma.getRunnable(null, async (session) => {

14    await Users.findOne({

15        session

16    });

17});
