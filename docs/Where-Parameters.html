linkWhere Parameters

An instance of the Where class can be used to easily create a statement and a bind parameter to be used in a query.

linkCreating a Where instance and getting its values

Creating a Where instance with values for the identifier n

1const where = new Where({

2    /* --> the node identifier is the key */

3    n: {

4        /* --> the per-identifier where values are used */

5        x: 5,

6        y: 'bar'

7    }

8});

9

10/* --> a statement can be generated to be used in the query */

11console.log(where.getStatement('text')); // "n.x = $x AND n.y = $y"

12/* --> the "bindParam" property has a BindParam instance whose values can be used in the query */

13console.log(where.bindParam.get()); // { x: 5, y: 'bar' }

Creating a Where instance with values for the identifiers n and o

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    },

6    o: {

7        z: true,

8    }

9});

10

11console.log(where.getStatement('text')); // "n.x = $x AND n.y = $y AND o.z = $z"

12console.log(where.bindParam.get()); // { x: 5, y: 'bar', z: true }

Since Where uses a BindParam, non-unique keys can be used which will be automatically associated with a unique key

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    },

6    o: {

7        /* --> since it's for a different identifier, we can use any key we want, even if it's used in another identifier */

8        x: true,

9    }

10});

11

12console.log(where.getStatement('text')); // "n.x = $x AND n.y = $y AND o.x = $x__aaaa"

13console.log(where.bindParam.get()); // { x: 5, y: 'bar', x_aaaa: true }

An existing BindParam instance can be used, to ensure unique keys. The same instance will be used in the Where instance, so it will be mutated.

1const existingBindParam = new BindParam({

2    x: 4,

3});

4

5const where = new Where(

6    {

7        n: {

8            /* --> the same key as in the bind param can be used */

9            x: 5,

10            y: 'bar'

11        }

12    },

13    existingBindParam

14);

15

16/* --> the "x" key already exists in the bind param, so a new one is used */

17console.log(where.getStatement('text')); // "n.x = $x__aaaa AND n.y = $y"

18console.log(where.bindParam.get()); // { x: 4, x_aaaa: 5, y: 'bar' }

19console.log(where.bindParam === existingBindParam); // true

An existing Where instance can be used. In this case, the parameters of it will be merged with the one that's being created. The existing Where instance won't be mutated at all.

1const existingWhere = new Where(

2    {

3        n: {

4            x: 4

5        },

6        o: {

7            z: true

8        }

9    }

10);

11

12console.log(existingWhere.getStatement('text')); // "n.x = $x AND o.z = $z"

13console.log(existingWhere.bindParam.get()); // { x: 4, z: true }

14

15const newWhere = new Where(

16    {

17        n: {

18            y: 'bar'

19        },

20        m: {

21            z: 'foo'

22        }

23    },

24    existingWhere

25);

26

27console.log(newWhere.getStatement('text')); // "n.x = $x AND n.y = $y AND o.z = $z AND m.z = $z__aaaa"

28console.log(newWhere.bindParam.get()); // { x: 4, y: 'bar', z: true, z__aaaa: 'foo' }

linkAdding parameters to an existing Where instance

Similarilly to creating a new Where instance, parameters can be added to an existing instance.

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    }

6});

7

8where.addParams({

9    n: {

10        z: true

11    },

12    o: {

13        x: 4

14    }

15});

16

17console.log(where.getStatement('text')); // "n.x = $x AND n.y = $y AND n.z = $z AND o.x = $x__aaaa"

18console.log(where.bindParam.get()); // { x: 5, y: 'bar', z: true, x__aaaa: 4 }

linkGetting the statement of a Where instance

A Where instance can easily be used in a query using its statement and bindParam properties

linkText form

This is how the statement can be used in a "text" form

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    },

6    o: {

7        z: true,

8    }

9});

10

11const textStatement = where.getStatement('text'); // n.x = $x AND n.y = $y AND o.z = $z

12const bindParamProperties = where.bindParam.get(); // { x: 5, y: 'bar', z: true }

13

14await queryRunner.run(

15    `MATCH (n), (o) WHERE ${textStatement} RETURN n, o`,

16    bindParamProperties

17);

linkObject form

This is how the statement can be used in an "object" form

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    },

6});

7

8const objectStatement = where.getStatement('object'); // { x: $x, y: $y }

9const bindParamProperties = where.bindParam.get(); // { x: 5, y: 'bar' }

10

11await queryRunner.run(

12    `MATCH (n ${objectStatement}) RETURN n`,

13    bindParamProperties

14);

This ignores the identifier, and is only available for the "equals" operator. So, it's recommended that it's used with only 1 identifier.

linkWhere operators

While some of the operators can be used with plain objects, some others need to use the exported Op variable. It contains symbols for the operators.

linkEquals

A direct object value corresponds to equality.

1const where = new Where({

2    n: {

3        x: 5

4    },

5});

6

7console.log(where.getStatement('text')); // n.x = $x

8console.log(where.getStatement('object')); // { x: $x }

9console.log(where.bindParam.get()); // { x: 5 }

Alternatively, the operator "eq" can be used.

1const where = new Where({

2    n: {

3        x: {

4            [Op.eq]: 5,

5        }

6    },

7});

8

9console.log(where.getStatement('text')); // n.x = $x

10console.log(where.getStatement('object')); // { x: $x }

11console.log(where.bindParam.get()); // { x: 5 }

linkAnd

The values of the parameters object are separated by an "and" operator

1const where = new Where({

2    n: {

3        x: 5,

4        y: 'bar'

5    },

6});

7

8console.log(where.getStatement('text')); // n.x = $x AND n.y = $y

9console.log(where.getStatement('object')); // { x: $x, y: $y }

10console.log(where.bindParam.get()); // { x: 5, y: 'bar' }

linkIn

1const where = new Where({

2    n: {

3        x: { 

4            [Op.in]: [1, 2, 3],

5        },

6        y: 2

7    },

8    o: {

9        z: {

10            [Op.in]: [4, 5, 6],

11        }

12    }

13});

14

15console.log(where.getStatement('text')); // n.x IN $x AND n.y = $y AND o.z IN $z

16// "object" statement not available

17console.log(where.bindParam.get()); // { x: [1, 2, 3], y: 2, z: [4, 5, 6] }

linkUsing a literal string

The class Literal can be used to use any given string in a Where condition.

1const where = new Where({

2    n: {

3        x: 2,

4        y: new Literal('n.x')

5    },

6    o: {

7        z: {

8            [Op.gt]: new Literal('n.y'),

9        }

10    }

11});

12

13console.log(where.getStatement('text')); // n.x = $x AND n.y = n.x AND o.z >= n.y

14// "object" statement not available

15console.log(where.bindParam.get()); // { x: 2 }

linkAcquire a Where instance

The acquire static can be used to ensure that a Where instance is at hand. If one is passed, it will be returned as is. If an object with where parameters is passed, a new Where instance will be created with them. Else, a new one will be created.

1const whereFirst = Where.acquire(null);

2console.log(whereFirst instanceof Where); // true

3

4const whereSecond = Where.acquire(whereFirst);

5console.log(whereFirst === whereSecond); // true

6

7/* --> an object with where parameters can be used */

8const whereWithPlainParams = Where.acquire({

9    n: {

10        x: 5

11    }

12});

13console.log(whereWithPlainParams instanceof Where); // true

14

15/* --> a BindParam instance can be passed to be used */

16const existingBindParam = new BindParam({ x: 4 });

17const whereWithBindParams = Where.acquire(

18    {

19        n: {

20            x: 5

21        }

22    },

23    existingBindParam

24);

25

26console.log(whereWithBindParams instanceof Where); // true

27console.log(whereWithBindParams.statement); // "n.x = $x__aaaa"

28console.log(whereWithBindParams.bindParam.get()); // { x: 4, x__aaaa: 5 }
