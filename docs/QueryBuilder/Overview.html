linkThe Query Builder

The QueryBuilder class can be used to easily generate cypher using objects. It supports all Neo4j clauses, and automatically uses a Bind Parameter for the given values.

linkCreating a QueryBuilder instance

After a QueryBuilder instance is created, parameters can be added to it by using the parameter methods. More info about them can be found in Clauses

1const queryBuilder = new QueryBuilder()

2    .match({

3        identifier: 'p1'

4    })

5    .return('p1');

6

7/* --> additional parameters can be added at any point */

8queryBuilder.limit(1);

linkAdding parameters by using an array of parameter objects

Additional parameters can be added using the addParams method of the QueryBuilder instance. The parameters are the same as in the constructor.

1// --> create the QueryBuilder instance using some initial parameters

2const queryBuilder = new QueryBuilder()

3    .match({

4        identifier: 'p1'        

5    });

6

7// --> add more parameters (using an array)

8queryBuilder.addParams([

9    {

10        match: '(n)'

11    }

12]);

13

14// --> add more parameters (using comma-separated objects)

15queryBuilder.addParams(

16    {

17        limit: 2

18    },

19    {

20        return: 'p1'

21    }

22);

linkGetting the statement from a QueryBuilder instance

The cypher statement of a QueryBuilder instance can be taken by using the getStatement method.

1const queryBuilder = new QueryBuilder().match({

2    identifier: 'p1'

3});

4

5console.log(queryBuilder.getStatement()); // MATCH (p1)

The BindParam of a QueryBuilder instance can be taken by using the getBindParam method.

1const queryBuilder = new QueryBuilder().match({

2    identifier: 'p1',

3    where: {

4        id: '1'

5    }

6});

7

8console.log(queryBuilder.getStatement()); // MATCH (p1 { id: $id })

9const bindParam = queryBuilder.getBindParam();

10console.log(bindParam.get()); // { id: '1' }

linkUsing an existing BindParam instance

In case a BindParam instance is used on the constructor, it will be used.

1const existingBindParam = new BindParam({

2    id: false,

3});

4

5const queryBuilder = new QueryBuilder(existingBindParam)

6    .match({

7        identifier: 'p1',

8        where: {

9            id: '1'

10        }

11    });

12

13console.log(queryBuilder.getStatement()); // MATCH (p1 { id: $id__aaaa })

14const bindParam = queryBuilder.getBindParam();

15console.log(bindParam === existingBindParam); // true

16console.log(bindParam.get()); // { id: false, id__aaaa: '1' }

linkRunning a QueryBuilder instance

A QueryBuilder instance can be run straight away, by providing a QueryRunner instance.

1/** let 'queryRunner' be a QueryRunner instance */

2const result = await new QueryBuilder()

3    .match('n')

4    .return('n')

5    .run(queryRunner);

6

7console.log(result.records.map(r => r.get('n')));

An existing session can be given

1    /** let 'queryRunner' be a QueryRunner instance and 'session' be a Session/Transaction */

2    await new QueryBuilder()

3        .raw('match n return n')

4        .run(queryRunner, session);

In order to avoid having to provide the QueryRunner instance on every call, the static queryRunner can be set. This can be done as soon as the Neogma instance is created, and should be set only once.

1    /** let 'neogma' be a Neogma instance */

2    QueryBuilder.queryRunner = neogma.queryRunner;

3

4    await new QueryBuilder()

5        .raw('match n return n')

6        .run();

An existing session can be given

1    /** assuming QueryBuilder.queryRunner is already set

2    /** let 'session' be a Session/Transaction */

3    await new QueryBuilder()

4        .raw('match n return n')

5        .run(session);
