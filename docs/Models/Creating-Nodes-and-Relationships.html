linkCreating Nodes and Relationships

Apart from building and saving an Instance, each model provides functions to directly create nodes and relationships.

linkCreating or a single node of the Model

1/* --> create a User node and get the Instance */

2const user = await Users.createOne(

3    /* --> the properties of the User node to be created */

4    {

5        id: '1',

6        name: 'John',

7        age: 38,

8    },

9    {

10        /* --> (optional, default true) validates the properties of the node */

11        validate: true,

12        /* --> (optional) an existing session or transaction to use */

13        session: null, // @see [Sessions](../Sessions)        

14        /* --> (optional) makes sure that the total relationships created by "where" parameters equal to 1. See below for examples. */

15        assertRelationshipsOfWhere: 1,

16    }

17);

18

19/* --> we can use the Instance as usual */

20console.log(user.name); // "John"

linkCreating a single node of the Model while relating it with other nodes

Neogma provides functionality for creating other nodes while creating a given node, and associating them automatically. Instead of creating the associated nodes, they can also be matched by a where clause. Everything runs in a single statement.

1const userWithOrder = await Users.createOne({

2    id: '1',

3    name: 'Alex',

4    /* --> associate with other nodes */

5    /* --> the Orders alias will be used, as defined in the Users model */

6    Orders: {

7        /* --> (optional) create new nodes and associate with them */

8        properties: [

9            /* --> creates the following 2 Order nodes, and creates a relationship with each one of them using the configuration of the Orders alias  */

10            {

11                id: '2'

12            },

13            {

14                id: '3',

15                items: 5,

16                /* --> the relationship is created with the following property. It uses the alias (Rating) as defined in the Model. The actual property property will be that in the Model relationships definition */

17                Rating: 4,

18                /* --> can create nodes and associate them with this Order node. The alias and configuration is that of the Orders model. This can be nested indefinitely */

19                /* --> the 'Critics' alias will be used, as defined in the 'Orders' model */

20                Critics: {

21                    properties: [{ id: '10' }]

22                }

23            }

24        ],

25        /* --> (optional) also associates the User node with existing Order nodes */

26        where: [

27            {

28                /* --> the Where clause find matching the existing Nodes */

29                params: { // @see [Where](../Where-Parameters)

30                    id: '3'

31                },

32                /* --> (optional) properties can be added to the relationship created by matching the User node with the existing Order nodes */

33                relationshipProperties: {

34                    Rating: 4,

35                },

36            },

37            {

38                /* --> another object can be used for matching the User node with the Order nodes of this where independently */

39                params: {

40                    items: 3,

41                }

42            }

43        ]

44    },

45    /* --> other aliases can be used here, to associate the User node with those of other Models */

46});

47

48console.log(userWithOrder.id); // "1"

linkCreating many nodes

For creating many nodes, the function createMany can be used, with identical parameters as createOne, with the only difference that the first param is an array of objects, instead of a plain object.

1const usersWithOrders = await Users.createMany(

2    [

3        {

4            id: '1',

5            name: 'John',

6        },

7        {

8            id: '2',

9            name: 'Alex',

10            Orders: {

11                /* --> same interface as createOne, which will apply only to this node */

12            }

13        }

14    ],

15    {

16        /* --> (optional, default true) validates all nodes */

17        validate: true,

18        /* --> (optional) an existing session or transaction to use */

19        session: null,

20    }

21);

22

23console.log(usersWithOrders[0].id); // "1"

24console.log(usersWithOrders[1].bar()); // "The name of this user is: Alex"

linkCreating relationships via the Model static

Relationships can be created via a Model static, by specifying params for both source and target nodes.

The following example created a relationship with the configuration of the alias Orders between the User nodes with name 'John' and the Order nodes with the id '2'.

1await Users.relateTo(

2    {

3        /* --> the alias of the relationship, as provided in the Model definition */

4        alias: 'Orders',

5        /* --> where parameters for the source and target nodes. Refer to the Where section for more information */

6        where: {

7            /* --> where parameters for the source node(s) */

8            source: {

9                name: 'John'

10            },

11            /* --> where parameters for the target node(s) */

12            target: {

13                id: '2',

14            },

15        },

16        /* --> properties of the relationship to be created */

17        properties: {

18            rating: 4,

19        },

20        /* --> (optional) throws an error if the created relationships are not equal to this number */

21        assertCreatedRelationships: 2,

22        /* --> (optional) an existing session or transaction to use */

23        session: null

24    }

25);

linkCreating relationships via the Instance method

Relationships can be created via an Instance method, by specifying params for just the target nodes. The source node will always be the one that corresponds to the instance, and its primary key field must be set.

The method relateTo is identical to the static, with the only difference being the where parameter, which now only refers to the target nodes.

1/* --> let 'user' be a Users instance */

2await user.relateTo(

3    {

4        /* --> the alias of the relationship, as provided in the Model definition */

5        alias: 'Orders',

6        /* --> where parameters for target node(s) */

7        where: {

8            id: '2',

9        },

10        /* --> properties of the relationship to be created */

11        properties: {

12            rating: 4,

13        },

14        /* --> (optional) throws an error if the created relationships are not equal to this number */

15        assertCreatedRelationships: 2,

16        /* --> (optional) an existing session or transaction to use */

17        session: null

18    }

19);
