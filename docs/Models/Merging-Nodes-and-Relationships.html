linkMerging Nodes and Relationships

Merging nodes and relationships happens with the same functions as creating nodes and relationships, by passing the appropriate params. Therefore, it is recommended that Creating Nodes and Relationships is known well.

  • When merging nodes, their data is used for the merge operation
    • i.e. MERGE (u:User { id: '1' })
  • When merging relationships, the merge is made on the source node, target node and relationship name, but the relationship properties get set afterwards
    • i.e. MERGE (u)-[r:CREATES]->(o) SET r.rating = 4

linkMerging a single or many nodes of a Model

For merging a single node, createOne can be used with the merge param set to true.

1/* --> merge a User node and get the Instance */

2const user = await Users.createOne(

3    /* --> the properties of the User node to be merged */

4    {

5        id: '1',

6        name: 'John',

7        age: 38,

8    },

9    {

10        /* --> by setting this to true, a MERGE query instead of a CREATE one will run */

11        merge: true,

12        /* --> (optional, default true) validates the properties of the node */

13        validate: true,

14        /* --> (optional) an existing session or transaction to use */

15        session: null,

16    }

17);

18

19/* --> we can use the Instance as usual */

20console.log(user.name); // "John"

For merging many nodes, createMany can be used with the merge param set to true.

1const users = await Users.createMany(

2    [

3        {

4            id: '1',

5            name: 'John',

6        },

7        {

8            id: '2',

9            name: 'Alex',

10        }

11    ],

12    {

13        /* --> by setting this to true, a MERGE query instead of a CREATE one will run */

14        merge: true,

15        /* --> (optional, default true) validates all nodes */

16        validate: true,

17        /* --> (optional) an existing session or transaction to use */

18        session: null,

19    }

20);

21

22console.log(usersWithOrders[0].id); // "1"

23console.log(usersWithOrders[1].bar()); // "The name of this user is: Alex"

linkCreating/Merging nodes and merging relationships with other nodes

When automatically associating with other nodes (either by creating them or by matching them), a MERGE instead of a CREATE can be used.

The following example uses createMany, but the same interface applies on createOne and instance save (when it doesn't exist in the database).

1const usersWithOrders = await Users.createMany(

2    [

3        {

4            id: '1',

5            name: 'John',

6        },

7        {

8            id: '2',

9            name: 'Alex',

10            /* --> associate with other nodes by their aliases */

11            /* --> the Orders alias will be used, as defined in the Users model */

12            Orders: {

13                /* --> (optional) configuration regarding what aspects of the 'properties' property to merge instead of create */

14                propertiesMergeConfig: {

15                    /* --> (optional) the created (Order) nodes will be merged, instead of created */

16                    nodes: true,

17                    /* --> (optional) the relationship between User and Order nodes will be merged, instead of created */

18                    relationship: true,

19                },

20                /* --> (optional) merge new nodes (as propertiesMergeConfig.nodes is true) and associate with them */

21                properties: [

22                    /* --> creates the following 2 Order nodes, and creates a relationship with each one of them using the configuration of the Orders alias  */

23                    {

24                        id: '2'

25                    },

26                    {

27                        id: '3',

28                        items: 5,

29                        /* --> the relationship is merged (as propertiesMergeConfig.relationship is true) with the following property (using its alias) */

30                        Rating: 4,

31                        /* --> can create (or merge) nodes and associate them with this Order node. The alias and configuration is that of the Orders model */

32                        /* --> the 'Critics' alias will be used, as defined in the 'Orders' model */

33                        Critics: {

34                            propertiesMergeConfig: {

35                                /* --> by setting this to false (or omitting it in the first place), the Critics nodes will be created, not merged */

36                                nodes: false,

37                                /* --> by setting this to false (or omitting it in the first place), the relationship between the Orders and the Critics nodes will be created, not merged */

38                                relationship: false,

39                            },

40                            properties: [{ id: '10' }]

41                        }

42                    }

43                ],

44                /* --> (optional) also associates the User node with existing Order nodes */

45                where: [

46                    {

47                        /* --> (optional) the relationship between the created User nodes and the matched Order nodes will be merged, instead of being created */

48                        merge: true,

49                        /* --> the Where clause find matching the existing Nodes */

50                        params: {

51                            id: '3'

52                        },

53                        /* --> (optional) properties can be added to the relationship merged by matching the User node with the existing Order nodes, using their alias */

54                        Rating: 5,

55                    },

56                    {

57                        /* --> another object can be used for matching the User node with the Order nodes of this where independently */

58                        params: {

59                            items: 3,

60                        },

61                        /* --> the realtionship of this match can be created, not merged */

62                        merge: false,

63                    }

64                ]

65            },

66            /* --> other aliases can be used here, to associate the User node with those of other Models */

67        }

68    ],

69    {

70        /* --> merges the root-level nodes (the Users - 'John', 'Alex') */

71        merge: true,

72        /* --> (optional, default true) validates all nodes */

73        validate: true,

74        /* --> (optional) an existing session or transaction to use */

75        session: null,

76    }

77);
