linkUsing ModelFactory

To define a Model, the information presented at Overview must be provided.

For the schema, revalidator is used. Its documentation is applied as-is for defining validating the properties of the nodes of this Model.

A sample Model definition with all configuration options is the following. Note that when using Typescript, to enable proper typing, some interfaces need to be provided.

1const Users = ModelFactory({

2    /* --> the label that the nodes of this Model have. For multiple labels, an array can be provided like ['User', 'New'] */

3    label: 'User',

4    /* --> The properties of the nodes of this Model and the validation for them. This follows the revalidator schema configuration */

5    schema: {

6        name: {

7            type: 'string',

8            minLength: 3,

9            required: true

10        },

11        age: {

12            type: 'number',

13            minimum: 0,

14        },

15        id: {

16            type: 'string',

17            required: true,

18        }

19    },

20    /* --> all the possible relationships (with other Models or itself), for relationship-related functions to work properly */

21    relationships: {

22        /* --> an arbitrary alias to be used for identifying this relationship when using the relationship-related functions */

23        Orders: {

24            /* --> reference to the Orders Model. For reference to this model, the value 'self' can be used */

25            model: Orders,

26            /* --> the direction of the relationship. Valid values are 'in' | 'out' | 'none' */

27            direction: 'out',

28            /* --> the name of this relationship */

29            name: 'CREATES',

30            /* --> properties of the relationship between the nodes */

31            properties: {

32                /* --> the key to be used that the property is a relationship property */

33                Rating: {

34                    /* --> the actual property to be created in the relationship */

35                    property: 'rating',

36                    /* --> schema validation for it */

37                    schema: {

38                        type: 'number',

39                    },

40                },

41            }

42        }

43    },

44    /* --> (optional) the key to be used as a unique identifier, which enables some Instance methods */

45    primaryKeyField: 'id',

46    /* --> (optional) statics to be added to the Model. In this example, can be called using `Users.foo()` */

47    statics: {

48        foo: () => {

49            return 'foo';

50        }

51    },

52    /* --> (optional) methods to be added to the Instance of this Model. In this example, they can be called on a Users Instance using `user.bar()` */

53    methods: {

54        bar: function() {

55            /* --> returns the id of this node */

56            return this.id;

57        }

58    }

59}, neogma); // <-- the neogma instance is used

60/* --> relationships can also be defined after the model definition, to avoid potential circular references */

61/* --> The same param as "relationships" is used */

62Users.addRelationships({

63    /* --> Orders can be defined either here or in the "relationships" param. Choose one! */

64    Orders: {...}

65});

linkUsing the Model's helpers

The created Model provides functions for database operations, as well as the following helpers

1    /* --> by providing an alias, gets the relationship configuration (model, direction, name, properties) */

2    Users.getRelationshipByAlias('Orders'); // --> { model: Orders, direction: 'out', name: 'CREATES', properties: {...} }

3    /** --> reverses the configuration of the relationship, so it's perfectly duplicated when definition the same relationship at the other model (here Orders) */

4    Users.reverseRelationshipByAlias('Orders');  // --> { model: Users, direction: 'in', name: 'CREATES', properties: {...} }

5    /* --> gets the primaryKeyField provided when defining the Model */

6    Users.getPrimaryKeyField(); // --> id

7    /* --> gets a name which is generated by the Model labels */

8    Users.getModelName(); // --> Users

9    /* --> getting a Model by its name by using the neogma instance */

10    neogma.modelsByName['Users'];
