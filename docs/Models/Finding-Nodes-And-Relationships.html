linkFinding Nodes

Neogma provides some basic functionality for matching, filtering, limiting and ordering nodes. For more complex find statements, one can use the driver for running a raw query.

linkFinding Many Nodes

1/* --> finds Users Nodes and returns an array of Users Instances */

2const users = await Users.findMany({

3    /* --> the where param for matching the Nodes */

4    where: {

5        /* --> the name property of the User Nodes must be 'John' and their id must be in: ('1', '2') */

6        name: 'John',

7        /* --> using the "in" Symbol from "Op" */

8        id: { [Op.in]: ['1', '2'] },

9    },

10    /* --> (optional) the limit of this query */

11    limit: 3,

12    /* --> (optional) the skip of this query */

13    skip: 3,

14    /* --> (optional) the order of this query, in this case by: age DESC, id ASC */

15    order: [['age', 'DESC'], ['id', 'ASC']],

16    /* --> (optional) throws NeogmaNotFoundError if no nodes are found (results length 0) */

17    throwIfNoneFound: true,

18    /* --> (optional) an existing session or transaction to use */

19    session: null,

20});

21

22console.log(users[0].bar()); // "The name of this user is: John"

23console.log(users[0].age, users[0].id); // 45 "2"

24console.log(users[1].age, users[1].id); // 45 "3"

25console.log(users[2].age, users[2].id); // 38 "1"

linkFinding a single Node

1/* --> finds a User Node and returns a Users Instances */

2const user = await Users.findOne({

3    /* --> the where param for matching the Node */

4    where: {

5        /* --> the name property of the User Node must be 'John' */

6        name: 'John',

7    },

8    /* --> (optional) the order of this query, in this case by: id ASC */

9    order: [['id', 'ASC']],

10    /* --> (optional) throws NeogmaNotFoundError if the node is not found */

11    throwIfNotFound: true,

12    /* --> (optional) an existing session or transaction to use */

13    session: null,

14});

15

16console.log(user.bar()); // "The name of this user is: John"

17console.log(user.id, user.age); // "1" 38

linkFinding nodes and relationships via the Model static

The findRelationships static can find related nodes via an Alias.

1const relationships = await Users.findRelationships({

2    /* --> the alias of the relationship, as provided in the Model definition */

3    alias: 'Orders',

4    /* --> (optional) where parameters for the query */

5    where: {

6        /* --> (optional) where parameters of the source node (i.e. User) */

7        source: {

8            id: '1'

9        },

10        /* --> (optional) where parameters of the target node (i.e. Order) */

11        target: {

12            id: '2'

13        },

14        /* --> (optional) where parameters of the relationship between the nodes */

15        relationship: {

16            rating: 4

17        },

18    },

19    /* --> (optional) limits the query. It's useful when the purpose is to find whether a relationship exists */

20    limit: 1,

21    /* --> (optional) minimum hops for a variable length relationship */

22    minHops: 1,

23    /* --> (optional) maximum hops for a variable length relationship. The value Infinity can be used for no limit on the max hops */

24    maxHops: 1,

25    /* --> (optional) an existing session or transaction to use */

26    session: null,

27});

28

29console.log(relationships[0]?.source.id); // '1'

30console.log(relationships[0]?.target.id); // '2'

31console.log(relationships[0]?.relationship.rating); // 4

linkFinding nodes and relationships via the Instance method

The findRelationships method can find related nodes via an Alias. It's a wrapper for the corresponding static, while using the Instance as the source node.

1/* --> let "user" be a Users instance with a primary field: id = '1' */

2const relationships = await user.findRelationships({

3    /* --> the alias of the relationship, as provided in the Model definition */

4    alias: 'Orders',

5    /* --> (optional) where parameters for the query */

6    where: {

7        /* --> (optional) where parameters of the target node (i.e. Order) */

8        target: {

9            id: '2'

10        },

11        /* --> (optional) where parameters of the relationship between the nodes */

12        relationship: {

13            rating: 4

14        },

15    },

16    /* --> (optional) limits the query. It's useful when the purpose is to find whether a relationship exists */

17    limit: 1,

18    /* --> (optional) minimum hops for a variable length relationship */

19    minHops: 1,

20    /* --> (optional) maximum hops for a variable length relationship. The value Infinity can be used for no limit on the max hops */

21    maxHops: 1,

22    /* --> (optional) an existing session or transaction to use */

23    session: null,

24});

25

26console.log(relationships[0]?.source.id); // '1'

27console.log(relationships[0]?.target.id); // '2'

28console.log(relationships[0]?.relationship.rating); // 4
