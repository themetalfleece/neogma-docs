linkInstances

Each Instance of a Model represents a node in the database, with its label, properties etc. matching those in the Model definition.

Instances offer flexible access to a node's properties, as well as operations regarding it, such as updating/deleting the node itself and its relationships.

linkCreating and saving an Instance

An Instance is returned from various Model operations (such as Create, Find), or a new Instance (which doesn't yet exist in the database) can be created as follows:

1    /* --> creates an Instance of the Users model, which still doesn't exist in the database */

2    const user = Users.build({

3        id: '1',

4        name: 'John',

5        age: 38,

6    });

7

8    /* --> the Instance can be saved to the database. This will run a CREATE operation to create a node to match the Users Model configuration (label etc.) */

9    await user.save({

10        /* --> (optional, default true) validates that the properties of the Instance are valid, given the schema of the Model definition */

11        validate: true,

12        /* --> (optional) an existing session or transaction to use */

13        session: null,

14    });

linkUpdating an Instance

1    /* --> given a user Instance, like the one created above, we can change the properties of the Instance */

2    user.name = 'Alex';

3

4    /* --> we reflect this change to the database. Since the node already exists in the databse, neogma will automatically run a MATCH-SET operation to update just the name of this node */

5    await user.save({

6        validate: false,

7    });

WARNING: Updating a instance ignores its Association fields (as described below) meaning that related nodes will not be created and associated. Only the initial save (when an instance doesn't yet exist in the database) creates related nodes.

linkAccessing the Instance's properties and methods

1    /* --> the Instance's properties and methods are accessible by their key */

2    console.log(user.id, user.name, user.age); // "1" "Alex" 38

3    /* --> all the instance properties can be taken as follows */

4    console.log(user.getDataValues()); // { id: "1", name: "Alex", age: 38 }

5    /* --> the methods, used in the Model definition can be used */

6    console.log(user.bar()); // "The name of this user is: Alex"

linkValidating an Instance

1    user.age = 30;

2    /* --> we can validate the properties of the Instance without saving it to the database. The properties of the Instance are valid, so this will not throw an error */

3    await user.validate();

4

5    try {

6        user.age = -1;

7        /* --> the properties of the Instance are invalid, so this will throw an error */

8        await user.validate();

9    } catch(err) { 

10        console.log(err); // NeogmaInstanceValidationError

11    }

linkCreating related nodes

1    /* --> by using the Related Nodes aliases specified in the Model definition, related nodes can also be created */

2    const userWithOrder = Users.build({

3        id: '2',

4        name: 'Alex',

5        /* --> the 'Orders' alias will be used, as defined in the 'Users' model */

6        Orders: {

7            /* --> (optional) create new nodes and associate with them */

8            properties: [{

9                /* --> a new Order node will be created with the following properties, and a relationship with the configuration of alias 'Orders' will be created (direction: out, name: CREATES) */

10                id: '3'

11            }],

12        }

13    });

14    /* --> when calling the following method, the User node, the Order node and the relationship between them */

15    await userWithOrder.save();

For more examples of creating realted nodes, refer to the Create operations, as the same interface is used
